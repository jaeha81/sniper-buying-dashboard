## 1. 제품 목표

### 1.1 한 줄 요약
> "마진이 검증된 해외 상품만 판매하는, 1인 운영 최적화 구매대행 플랫폼"

### 1.2 배경 및 문제 정의
- 국내 소비자는 해외 직구에 관심이 있으나 통관, 배송, 언어 장벽으로 망설임
- 구매대행 운영자는 상품 수익성 검증 없이 직관으로 소싱 → 마진 악화
- 기존 구매대행 플랫폼(몰테일, 배대지)은 물류 중심, 상품 큐레이션·수익관리 도구 없음

### 1.3 목표
1. **소비자**: 언어·통관·배송 부담 없이 안전하게 해외 상품 구매 경험 제공
2. **운영자(자신)**: Sniper Score 기반 수익성 검증 + 관리자 도구로 1인 운영 효율화
3. **사업**: Phase 1 MVP 3개월 내 월 순마진 50만 원 달성

### 1.4 측정 기준 (성공 지표 / KPI)

| 지표 | Phase 1 목표 | Phase 2 목표 |
|------|------------|------------|
| 월 주문 수 | 20건 | 100건 |
| 월 순마진 | 50만 원 | 300만 원 |
| 평균 마진율 | 30% 이상 | 35% 이상 |
| 재구매율 | - | 20% 이상 |
| 상품 등록 수 | 30개 | 150개 |
| Sniper Score 평균 | 75점 이상 | 80점 이상 |

## 2. 타겟 사용자

### 2.1 주요 타겟 (B2C — 쇼핑몰 고객)
- **페르소나**: 28~42세 직장인, 해외직구 경험 1회 미만 또는 없음
- **니즈**: 좋은 해외 제품을 사고 싶지만 영어, 관세, 배송 복잡성이 두려움
- **행동 패턴**: 네이버/인스타 검색 → 상세페이지 신뢰 확인 → 결제
- **지불 의사**: 국내가 대비 5~15% 추가 비용 수용 (편의성 프리미엄)

### 2.2 운영자 (B2B-Self — 관리자 대시보드 사용자)
- 본인(1인 사업자)이 유일한 관리자
- 상품 소싱, 마진 계산, 주문 처리, CS 대응 모두 담당
- 추후 Phase 3에서 타 구매대행자에게 SaaS로 판매 가능

## 3. 핵심 기능 목록

### 3.1 고객 화면 (쇼핑몰)

#### F-C01: 메인 홈 (`/`)
- 히어로 배너: 구매대행 신뢰 포인트 3가지 강조
- 추천 상품 섹션: Sniper Score 상위 6개 상품 카드 노출
- 카테고리 퀵 링크: 건강식품 / 운동용품 / 뷰티 / 아웃도어 / 전자기기
- 배송 프로세스 인포그래픽 (소싱 → 국제배송 → 통관 → 국내배송)
- 신뢰 지표: 누적 주문 건수, 5점 리뷰 샘플, 안전 결제 배지

#### F-C02: 상품 목록 (`/products`)
- 카테고리 필터 (다중 선택)
- 가격 범위 슬라이더
- 정렬: Sniper Score 순 / 가격 낮은순 / 인기순 / 최신순
- 상품 카드: 썸네일, 상품명, 원산지 국가 플래그, 국내 판매가, 마진율 배지(운영자만)
- 무한 스크롤 또는 페이지네이션 (20개/페이지)
- 검색창: 상품명 실시간 검색

#### F-C03: 상품 상세 (`/products/[id]`)
- 상품 이미지 갤러리 (슬라이더)
- 상품명 (한국어), 원산지, 소싱 플랫폼(선택적 노출)
- 가격: 국내 판매가 (원화), 예상 총비용 상세 펼치기
- 예상 배송 일정: 소싱 기간 + 국제배송 + 통관 + 국내배송
- 통관 안내: 해당 카테고리 주의사항 (식품/화장품 등)
- 상세 설명 섹션: 제품 특징, 사용법, 성분(해당 시)
- 구매 버튼: 장바구니 담기 / 바로 구매
- 관련 상품 추천 (같은 카테고리, 4개)

#### F-C04: 장바구니 (`/cart`)
- 담긴 상품 목록 (썸네일, 이름, 수량, 단가, 소계)
- 수량 변경 (+/-), 삭제
- 주문 요약: 상품합계, 국내배송비, 예상 합계
- "결제하기" 버튼 → `/checkout`

#### F-C05: 결제 (`/checkout`)
- 주문자 정보 입력: 이름, 연락처, 이메일
- 배송지 정보 입력: 주소 검색(다음 API), 상세주소
- 주문 상품 최종 확인
- 결제 수단 선택 (Phase 2: 토스페이먼츠 / Phase 1: 입금 안내)
- 개인정보 수집 동의, 구매대행 이용약관 동의 (필수)
- "주문 완료" 버튼

#### F-C06: 주문 완료 (`/order-complete`)
- 주문번호, 예상 처리 일정 안내
- 카카오채널/이메일 알림 수신 안내
- "쇼핑 계속하기" 버튼

#### F-C07: 배송 가이드 (`/shipping-guide`)
- 국제배송 흐름 단계별 설명
- 예상 소요 기간 (국가별 표)
- 통관 기본 안내 (목록통관 vs 일반통관)
- 분실/파손 대응 정책

#### F-C08: FAQ (`/faq`)
- 아코디언 형식 자주묻는질문 20개 이상
- 카테고리별 분류: 주문/결제, 배송, 통관, 환불/교환, 기타

#### F-C09~C11: 정책 페이지
- `/policy/privacy`: 개인정보처리방침
- `/policy/terms`: 이용약관 (구매대행 특화)
- `/policy/refund`: 환불/교환 정책 (해외 구매대행 특성 반영)

### 3.2 관리자 화면 (Admin Dashboard)

> 접근 제한: Supabase Auth 이메일 로그인 + `admin` role 확인

#### F-A01: 관리자 홈 대시보드 (`/admin`)
- 오늘/이번주/이번달 주문 건수, 매출, 순마진 KPI 카드
- 마진율 추이 라인 차트 (최근 30일)
- 카테고리별 매출 파이 차트
- 처리 대기 주문 목록 (빠른 접근)
- 최근 등록 상품 5개
- Make.com 자동화 최근 실행 상태

#### F-A02: 상품 관리 (`/admin/products`)
- 등록된 상품 테이블: ID, 상품명, 카테고리, 판매가, 재고상태, Sniper Score, 상태(판매중/품절/숨김)
- 상품 추가/수정/삭제 (CRUD)
- 상품 이미지 업로드 (Supabase Storage)
- 상품별 판매 통계 요약

#### F-A03: 상품 후보 (`/admin/product-candidates`)
- 소싱 검토 중인 상품 후보 목록
- Sniper Score 자동 계산: 각 항목 점수 입력 → 총점 실시간 표시
- 항목별 점수 슬라이더/입력 (국내수요, 해외가격경쟁력, 마진율, 배송안정성, 통관리스크, 경쟁강도, 상세페이지설득력, 자동화적합도)
- 70점 이상이면 "등록 권장" 배지 + 실제 상품으로 이동 버튼
- 소싱 URL, 해외가격, 메모 필드

#### F-A04: 주문 관리 (`/admin/orders`)
- 주문 목록 테이블: 주문번호, 주문일, 고객명, 상품, 금액, 상태
- 주문 상태 변경: 접수확인 → 결제확인 → 소싱중 → 국제배송중 → 통관중 → 국내배송중 → 완료
- 주문 상세: 주문 상품, 고객 정보, 배송지, 결제 정보
- 상태 변경 시 고객 이메일 자동 발송 (Make.com 트리거)
- 엑셀 내보내기 (월별)

#### F-A05: 고객 관리 (`/admin/customers`)
- 고객 목록: 이름, 이메일, 가입일, 총 주문 수, 총 결제액
- 고객 상세: 주문 이력, 배송지 목록
- 블랙리스트 처리 기능

#### F-A06: 마진 계산기 (`/admin/margins`)
- **핵심 기능**: 상품별 비용 항목 입력 → 순마진/마진율 실시간 계산
- 입력 항목:
  - 해외원가 (외화 입력 + 실시간 환율 적용)
  - 현지배송비, 국제배송비
  - 관세 예상액 (카테고리별 관세율 자동 적용 또는 직접 입력)
  - 부가세 예상액 (자동 계산)
  - 국내배송비, 결제수수료(%), 기타비용
  - 국내판매가 (목표 마진율 역산 기능 포함)
- 결과:
  - 총원가, 예상순마진, 마진율(%) 강조 표시
  - 마진율 색상 코드: 빨강(<20%) / 노랑(20~30%) / 초록(30%+)
- 계산 이력 저장 (Supabase) 및 불러오기

#### F-A07: 자동화 로그 (`/admin/automation-logs`)
- Make.com 웹훅 수신 이력 표시
- 시나리오별 실행 결과: 성공/실패/스킵
- 환율 업데이트 이력
- 재고 체크 이력
- 오류 발생 시 알림 방법 설정

#### F-A08: 설정 (`/admin/settings`)
- 환율 설정 (USD/JPY/EUR/CNY → KRW, 수동 업데이트 또는 Make.com 자동)
- 카테고리 관세율 테이블 관리
- 배송비 기본값 설정
- 결제수수료율 설정
- 이메일 알림 템플릿 관리
- Make.com 웹훅 URL 표시 및 재생성

## 4. 비기능 요구사항

### 4.1 성능
- 상품 목록 페이지: 초기 로드 LCP < 2.5초 (Vercel Edge 캐싱 활용)
- 마진 계산기: 입력 후 결과 표시 < 100ms (클라이언트 사이드 계산)
- Supabase 쿼리: 단순 조회 < 200ms

### 4.2 보안
- 관리자 라우트 `/admin/**`: Supabase Auth 미들웨어로 보호
- API Routes: 서비스 롤 키는 서버 사이드에서만 사용
- 환경변수: `.env.local`에만 저장, 절대 커밋 금지
- SQL Injection: Supabase SDK의 파라미터화 쿼리 사용

### 4.3 접근성
- WCAG 2.1 Level AA 준수 (shadcn/ui 기본 지원)
- 모바일 퍼스트 반응형 디자인 (320px ~ 1440px)
- 키보드 내비게이션 지원

### 4.4 유지보수성
- TypeScript strict 모드 활성화
- ESLint + Prettier 코드 스타일 통일
- 컴포넌트 단위 기능 분리
- Supabase 마이그레이션 파일로 DB 변경 이력 관리

| 라이브 채팅 | 카카오채널로 대체 |
| AI 추천 엔진 | Phase 2~3 검토 |

## 7. 기술 아키텍처 개요

```
[브라우저]
    ↓ HTTPS
[Vercel Edge Network]
    ↓
[Next.js App Router]
    ├── SSR/SSG 페이지 → Supabase PostgreSQL
    ├── API Routes → Supabase Admin SDK
    └── Static Assets → Supabase Storage
    
[Make.com]
    ├── 환율 업데이트 Webhook → /api/exchange-rate
    ├── 재고 체크 → /api/stock-check
    └── 주문 알림 → 이메일/카카오
```

## 8. 용어 정의

| 용어 | 정의 |
|------|------|
| Sniper Score | 상품 수익성·판매 가능성을 100점 만점으로 평가하는 독자 지표 |
| 목록통관 | 150달러 이하 개인용 수입품에 적용되는 간소화 통관 방식 |
| 일반통관 | 150달러 초과 또는 상업용 수입품에 적용되는 정식 통관 |
| 소싱 | 해외 플랫폼에서 판매할 상품을 구매하는 행위 |
| 드롭쉬핑 | 재고 없이 주문 후 해외에서 직접 고객에게 배송하는 방식 |
